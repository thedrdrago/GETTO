## THE PROBLEM

Italian construction projects—especially in Sicily—run on WhatsApp chaos. A typical mid-size residential project has 10+ WhatsApp groups, with drawings scattered across message threads, email attachments, printed sheets, and old PDFs. There is **no version control**, no traceability, and no single source of truth.

**Real consequences:** A Sicilian plumber executed a 2-year-old bathroom layout drawing, resulting in €47k rework. Version confusion is endemic. Comments and instructions are scattered across WhatsApp, site visits, and phone calls—untrackable and unactionable. Current tools (email, WeTransfer, cloud storage) are not designed for **managing drawings as operational assets during execution**. ERPs assume discipline and top-down control that small-mid firms don't have.

The problem costs Italian construction **€billions annually** in rework, delays, and disputes. Nobody admits it because it's treated as "normal."

1. **Mobile-first reality:** 87% of Italian builders access project info via smartphone; desktop tools are irrelevant.
2. **WhatsApp dominance = distribution channel:** 94% of construction teams already use WhatsApp daily; JETTO piggybacks on this network, not disrupts it.
3. **Cheap cloud storage + AI:** Infrastructure costs dropped 10x in 5 years, enabling a profitable free tier. Future dataset insights unlock long-term value.

Post-pandemic, teams expect digital collaboration but **refuse complex software**. JETTO is the middle ground: simple, WhatsApp-native, and purpose-built for drawings.

## THE PRODUCT

**JETTO is a centralized drawing vault for construction execution.**  
**It is NOT an ERP, project management suite, procurement tool, or BIM collaboration platform.**

**What JETTO Does:**
- Single source of truth for all project drawings (PDFs, DWGs, images)
- Version control: newest supersedes old; archive hidden but accessible via "See older versions"
- Comment threads anchored to specific drawings + versions
- Role-based access: Architects upload/revise, PMs enforce, Builders execute, Owners view
- Notifications (in-platform + email + optional WhatsApp): new upload, revision, comment, ready-for-execution

**What JETTO Does NOT Do:**
- Project chat (use WhatsApp)
- Procurement, invoicing, time tracking (use Excel/existing ERP)
- BIM/3D collaboration (use Autodesk)
- Change how builders work (we overlay workflows, not replace them)

**Scope discipline is the strategy.** JETTO solves one problem completely: ensuring the right drawing is always accessible, current, and traceable.

---

## HOW IT WORKS

**The 6-Step Loop (repeated 50–100× per project):**

1. **Upload:** Architect uploads drawing; status = "Draft"
2. **Comment:** Builder marks issue directly on PDF: "Rebar spacing unclear"
3. **Revise:** Architect uploads new version; old version auto-archived
4. **Ready:** PM reviews, marks "Ready for execution"; notification sent
5. **Execute:** Builder on-site marks "Executed" with timestamp
6. **Archive:** Drawing locked; comments preserved; full audit trail

Every action is logged. Every version is retrievable. Role-based dashboards surface priorities (e.g., builders see "Ready for Execution" first; architects see "Awaiting Revision" first).
