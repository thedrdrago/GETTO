If you want to chat, use WhatsApp. If you want to invoice, use your existing tools. JETTO sits in the middle, ensuring the right drawing is always accessible, always current, always traceable. ## SLIDE 5: HOW IT WORKS—THE 6-STEP LOOP

**From Upload to Execution in One Clean Cycle**

1. **Upload**: Architect uploads drawing (PDF/DWG); sets status "Draft"
2. **Comment**: Builder marks issue directly on PDF: "Rebar spacing unclear here"
3. **Revise**: Architect uploads new version; old archived automatically
4. **Ready**: PM reviews, marks "Ready for execution"; notification sent
5. **Execute**: Builder on-site, marks drawing "Executed" with timestamp
6. **Archive**: Drawing locked; comments preserved; audit trail complete

**All actions logged. All versions retrievable. All roles see their priorities first.**

**Speaker Notes:**  
"This loop happens 50–100 times per mid-size project. Every drawing goes through it. The magic is that JETTO makes this loop visible, traceable, and friction-free. When a builder opens the app on-site, they see 'Ready for Execution' drawings at the top. When an architect logs in, they see 'Awaiting Revision' drawings first. Role-based intelligence, not complex UI. And because every action is logged, you have a full audit trail for disputes, insurance claims, or regulatory reviews. The loop is the product."
