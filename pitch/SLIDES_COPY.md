## SLIDE 4: PRODUCT

**A Centralized Drawing Vault for Execution**
**NOT an ERP, PM Suite, or BIM Collaboration Tool**

**What JETTO Does:**
• Single source of truth for all project drawings (PDFs, DWGs, images)
• Comment threads anchored to specific drawings + versions
• Version control: newest supersedes old; archive hidden but accessible
• Role-based access: Architect uploads/revises, PM enforces, Builders execute, Owners view
• Notifications in-platform + email + optional WhatsApp ping

**What JETTO Does NOT Do:**
• Project chat (use WhatsApp) • Procurement, invoices, time tracking (use Excel/ERP) • BIM/3D collaboration (use Autodesk) • Change workflows (we overlay, not replace)

**Italian:** Un cassetto per i disegni, non un software gestionale. Copre il buco che WhatsApp ed email non possono coprire.

---

## SLIDE 5: HOW IT WORKS

**From Upload to Execution in One Clean Cycle**

1. Upload: Architect uploads drawing (PDF/DWG); sets status "Draft"
2. Comment: Builder marks issue directly on PDF: "Rebar spacing unclear here"
3. Revise: Architect uploads new version; old archived automatically
4. Ready: PM reviews, marks "Ready for execution"; notification sent
5. Execute: Builder on-site, marks drawing "Executed" with timestamp
6. Archive: Drawing locked; comments preserved; audit trail complete

All actions logged. All versions retrievable. All roles see their priorities first.
